我要开发一款网页足球经理游戏的 MVP。

目标：
玩家可以管理一支球队，在包含 2 级联赛的体系中完成多轮比赛。每级联赛 20 支球队，赛季结束后前 3 名升级、后 3 名降级。新游戏开始时，玩家球队从最低级联赛中随机挑选。

核心闭环：
1. 查看球队球员
2. 自动按球员能力选择首发阵容
3. 模拟一场比赛
4. 查看比分
5. 查看积分榜
6. 进入下一轮

第一版不要做：
- 真实球员授权
- 转会市场
- 青训
- 复杂财政
- 3D/2D 实时比赛画面
- 联网对战

需要设计：
1. 项目目录结构
2. 数据模型：Player、Team、Match、League、Standing
3. 比赛模拟规则
4. 页面结构
5. 开发里程碑
6. 每个里程碑的验收标准
7. 后续可扩展方向
8. 两级联赛、升降级与新赛季流转规则

技术倾向：
- 前端网页游戏
- TypeScript
- 初期可用 JSON 或本地存储
- 后续可接后端数据库
