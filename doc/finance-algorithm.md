# 球员工资、身价及俱乐部收入算法

## 一、球员身价算法 (calculateMarketValue)

### 计算公式

```
基础价值 = overall² × 100
潜力奖励 = MAX(0, potential - overall) × 5000
年龄调整系数 = 根据年龄区间确定
最终价值 = MAX(1000, ROUND((基础价值 + 潜力奖励) × 年龄调整系数))
```

### 年龄调整系数表

| 年龄区间 | 调整系数 |
|---------|---------|
| ≤ 21岁 | 1.25 |
| 22-25岁 | 1.15 |
| 26-29岁 | 1.0 |
| 30-32岁 | 0.75 |
| > 32岁 | 0.45 |

### 算法说明

- 基础价值与球员能力值的平方成正比，能力值越高，身价增长越快
- 潜力奖励仅在潜力高于当前能力时生效，激励培养年轻有潜力球员
- 年龄调整体现了足球市场的普遍规律：26-29岁是球员黄金期，超过32岁后身价大幅下降
- 最低身价为1000，避免出现负数或零身价

## 二、球员工资算法 (calculateWeeklyWage)

### 计算公式

```
周薪 = CLAMP(ROUND(marketValue × 0.002), 100, 200000)
```

### 参数说明

| 参数 | 值 | 说明 |
|-----|---|-----|
| 系数 | 0.002 | 周薪约为身价的0.2% |
| 最小值 | 100 | 最低周薪 |
| 最大值 | 200000 | 最高周薪 |

### 算法说明

- 周薪直接基于球员当前市场价值计算
- 线性比例关系，身价越高周薪越高
- 设定上下限防止极端值

## 三、俱乐部收入算法

### 3.1 门票收入 (calculateTicketIncome)

#### 计算公式

```
上座率 = CLAMP(0.45 + fanBase / 200, 0.35, 1)
到场人数 = ROUND(stadiumCapacity × 上座率)
门票收入 = 到场人数 × ticketPrice
```

#### 参数说明

| 参数 | 说明 |
|-----|-----|
| fanBase | 俱乐部球迷基数 |
| stadiumCapacity | 球场容量 |
| ticketPrice | 门票单价 |
| 最低上座率 | 0.35 (35%) |
| 最高上座率 | 1 (100%) |

#### 算法说明

- 基础上座率为45%，球迷基数越高，上座率越高
- 上座率受球场容量和球迷基数共同影响
- 设定上座率范围在35%-100%之间

### 3.2 财务结算流程 (settleMatchFinances)

#### 结算步骤

1. **初始化**：创建球队收支映射、财务日志数组和结算摘要

2. **门票收入结算**：遍历所有已完成比赛，为主场球队计算并记录门票收入

3. **工资支出结算**：遍历所有球队，计算该队所有球员的周薪总和作为支出

4. **更新球队余额**：将收支净额应用到球队账户余额

#### 数据结构

**FinanceSummary（财务摘要）**：
- `ticketIncome`: 门票收入总额
- `wageExpense`: 工资支出总额
- `net`: 净收支
