# 球员能力值金字塔分布 - 产品需求文档

## 概述
- **Summary**: 为6级联赛共3000名球员建立金字塔式能力值分配系统，高级联赛球员能力高但人数权重低，低级联赛球员能力低但人数权重高。
- **Purpose**: 明确各联赛的实力差距，使球员能力分布呈现"金字塔"形态，提升联赛升级/降级的真实感和挑战性。
- **Target Users**: miniFM 游戏的玩家（俱乐部经理）

## 目标
- 按金字塔分布（权重 1:4:9:16:25:36）分配各联赛能力区间
- 第1级联赛球员能力最高，第6级联赛能力最低
- 能力值范围严格落在 [1, 200] 区间
- 各联赛能力区间不重叠，形成明显分层
- 平均能力区间跨度约 33，覆盖 1-200 的全范围

## 非目标（范围外）
- 不调整单球队内球员能力差异（仍在联赛范围内随机生成）
- 不调整球员年龄分布逻辑
- 不调整身价/周薪公式（保持现有的 `ability * 1000 * ageFactor`）
- 不调整比赛引擎中基于 ability 的计算

## 背景与上下文
当前 `DataGenerator.generatePlayer()` 使用简单线性能力范围：
- 第1级联赛: 120-180
- 第6级联赛: 50-100

问题：能力范围有重叠（如第1级的低端 120 与第2级的高端重叠），无法体现金字塔分布。能力越高的球员应该越稀少。

## 功能需求
- **FR-1**: 在 CONFIG 中定义各联赛能力范围常量（`LEAGUE_ABILITY_RANGES`）
- **FR-2**: 修改 `generatePlayer()` 从新的能力范围生成球员能力值
- **FR-3**: 球队生成时根据 `leagueLevel` 从对应能力范围生成球员

## 非功能需求
- **NFR-1**: 代码修改仅涉及能力范围，不破坏其他逻辑
- **NFR-2**: 配置表清晰、易读，便于后续调整

## 约束
- 能力值必须为整数 [1, 200]
- 6 级联赛，每级 20 队，每队约 20-25 人
- 不引入新依赖

## 假设
- 能力区间严格非重叠，相邻级别之间存在约 1-2 点的空隙或正好相连（简化起见使用连续划分）
- 金字塔权重近似 n² 分布

## 能力范围设计

| 联赛级别 | 能力范围 (min-max) | 区间宽度 | 联赛内部分布 | 说明 |
|---------|-------------------|---------|------------|------|
| L1（顶级） | 151 - 200 | 50 | 金字塔（平方反比权重） | 精英球员，高能力值稀有 |
| L2 | 121 - 170 | 50 | 金字塔（平方反比权重） | 准精英 |
| L3 | 91 - 140 | 50 | 金字塔（平方反比权重） | 中高级 |
| L4 | 61 - 110 | 50 | 金字塔（平方反比权重） | 中级 |
| L5 | 31 - 80 | 50 | 金字塔（平方反比权重） | 初级 |
| L6（最低级） | 1 - 50 | 50 | 金字塔（平方反比权重） | 业余球员，高能力值稀有 |

**设计说明**：
- 相邻联赛范围有 30 个值的重叠（L1 151-170 与 L2 151-170 重叠等），使球员分布平滑
- 每级联赛内部使用金字塔分布：能力值 = max 的权重为 1²，能力值 = min 的权重为 50²
- 因此高能力值球员在同联赛中人数最少，呈金字塔形态
- 联赛平均值 L1 > L2 > L3 > L4 > L5 > L6，实现级别间的实力差距

**验证（1000 样本/级）**：
- L1 平均能力 ≈ 163，L2 ≈ 133，L3 ≈ 103，L4 ≈ 73，L5 ≈ 43，L6 ≈ 13
- 各级高端 25% 区间球员占比仅约 2%（均匀分布下应为 25%）✓
- 严格递减 ✓
- 能力越高球员越少 ✓

## 金字塔分布算法

使用离散累积权重法（平方反比）：
- 对联赛范围 `[min, max]`，设 `N = max - min + 1`
- 能力值 `max - i + 1` 的权重 = `i²`（i 从 1 到 N）
- 总权重 = `N*(N+1)*(2N+1)/6`
- 在 `[1, totalWeight]` 内均匀取一个值，反向累加找到对应能力值

## 接收标准

### AC-1: 第1级联赛球员能力落在 [167, 200]
- **Given**: 玩家在第1级联赛查看球队球员
- **When**: 查看任意第1级联赛球队球员列表
- **Then**: 所有球员的能力值在 167-200 之间
- **Verification**: programmatic
- **Notes**: 通过采样 100 名随机生成的 L1 球员验证

### AC-2: 第6级联赛球员能力落在 [1, 34]
- **Given**: 玩家在第6级联赛（最低级别）
- **When**: 查看任意第6级联赛球队球员列表
- **Then**: 所有球员的能力值在 1-34 之间
- **Verification**: programmatic

### AC-3: 各联赛能力区间互不重叠且完全覆盖 [1, 200]
- **Given**: 所有 6 个联赛级别都存在
- **When**: 收集各联赛能力范围
- **Then**: 范围无重叠且合并后为 [1, 200]
- **Verification**: programmatic

### AC-4: 能力越高的球员在全联赛中越稀少（金字塔验证）
- **Given**: 生成所有 3000 名球员
- **When**: 统计各能力区间的球员数量
- **Then**: 能力值高的区间球员数远低于能力值低的区间（L1 平均权重为 L6 的 1/36）
- **Verification**: programmatic

### AC-5: 球员生成在能力区间内均匀分布
- **Given**: 大量随机生成的同一联赛球员
- **When**: 计算该联赛内球员能力平均值
- **Then**: 平均值应接近该联赛能力区间中点（误差在 ±3 内）
- **Verification**: programmatic

### AC-6: 代码可读性：能力范围集中定义在 CONFIG 中
- **Given**: 新开发者阅读 core.js
- **When**: 查找能力范围配置
- **Then**: 在 CONFIG 对象中能找到清晰的 LEAGUE_ABILITY_RANGES 配置表
- **Verification**: human-judgment
